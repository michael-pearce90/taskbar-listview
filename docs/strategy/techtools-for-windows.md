# TechTools for Windows Strategy

## Identity

Display name: **TechTools for Windows**.

Description target: **Technician-focused open-source tools for Windows.**

Target repository slug: `techtools-for-windows`.

TechTools for Windows is not affiliated with or endorsed by Microsoft.
Windows is used descriptively for compatibility only.

## Direction

The suite should collect small Windows tools that are useful to technicians
because they are narrow, auditable, reversible, and honest about support.
Modules should solve specific operational pain points rather than become a
general shell replacement or broad customisation platform.

The first module is Taskbar ListView, currently documentation and research
tooling only. Browser Choice Tools is a proposed future module with no code
yet.

## Public Claims

Public wording must distinguish:

- proposal from implementation;
- inventory evidence from validated support;
- local logging from telemetry;
- compatibility description from Microsoft affiliation; and
- module-specific risk from suite-level identity.

Do not use product logos or service marks for Microsoft, Windows, Edge, Bing,
PowerToys, or Sysinternals. Do not claim this is a Microsoft project.
